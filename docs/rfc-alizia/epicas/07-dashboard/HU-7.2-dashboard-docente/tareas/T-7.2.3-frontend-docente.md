# T-7.2.3: Frontend del dashboard docente

**Historia:** HU-7.2 — Dashboard docente
**Tipo:** Frontend
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripción

Implementar la vista de dashboard para el rol docente con los 4 widgets: próximas clases, estado de planificaciones, documentos publicados y clases compartidas.

## Implementación

### Layout

Grid de 2x2 con los widgets:

```
┌──────────────────────┬──────────────────────┐
│ Próximas clases      │ Mis planificaciones  │
│ (timeline + estados) │ (barras por materia) │
├──────────────────────┼──────────────────────┤
│ Documentos           │ Clases compartidas   │
│ (lista + badge nuevo)│ (lista con docente)  │
└──────────────────────┴──────────────────────┘
```

### Componentes

```javascript
// Widget: Próximas clases
function renderUpcomingClassesWidget(classes) {
    // Timeline vertical con:
    // - Fecha y día de la semana
    // - Horario
    // - Materia + curso
    // - Título de la clase
    // - Badge de estado: ✅ planned, ⬜ pending
    // - Badge 🤝 si es compartida
    // - Click → navegar a planificación
    // - Clase sin planificar próxima (< 3 días) → highlight urgente
}

// Widget: Mis planificaciones
function renderPlanningStatusWidget(status) {
    // Por cada materia+curso:
    // - Nombre de materia (curso)
    // - Barra de progreso (planned/total)
    // - Texto "8/10 clases"
    // - Click → navegar al cronograma
}

// Widget: Documentos publicados
function renderDocumentsWidget(documents) {
    // Lista de documentos:
    // - Nombre + área
    // - Badge 🆕 si is_new
    // - Fecha de publicación
    // - Click → navegar al documento (y registrar vista)
}

// Widget: Clases compartidas
function renderSharedClassesWidget(classes) {
    // Lista de próximas clases compartidas:
    // - Fecha + horario
    // - Mi materia 🤝 otra materia
    // - Nombre del otro docente
    // - Estado de planificación
    // - Click → navegar a planificación
}
```

### Carga de datos

```javascript
async function loadTeacherDashboard() {
    state.dashboardLoading = true;
    const data = await fetch('/dashboard/teacher').then(r => r.json());
    state.dashboard = data;
    state.dashboardLoading = false;
    renderDashboard();
}
```

## Notas

- Responsive: en mobile los widgets se apilan en una sola columna
- Las clases urgentes (sin planificar, próximas en < 3 días) se destacan con borde rojo
- El badge "nuevo" desaparece después de que el docente abre el documento
- Si el docente no tiene materias asignadas, mostrar empty state con mensaje informativo
- Los mismos estilos base (`.dashboard-grid`, `.dashboard-widget`) se reutilizan del dashboard coordinador (T-7.1.2)
