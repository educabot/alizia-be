# T-4.5.1: Usecase PublishDocument

**Historia:** HU-4.5 — Publicacion y estados
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripcion

Implementar el usecase `PublishDocument` que transiciona un documento de coordinacion de estado `in_progress` a `published`. Antes de publicar, se ejecutan validaciones: secciones requeridas son bloqueantes (422), sub-topics no asignados son **warning confirmable** (el usuario puede forzar con `force: true`). Basado en decisiones P12, P13.

## Implementacion

### Ubicacion

`src/core/usecases/coordination/publish_document.go`

### Request

```go
type PublishDocumentRequest struct {
    OrgID  uuid.UUID
    DocID  uint
    Force  bool // Si true, ignora warnings (sub-topics sin asignar)
}
```

### Validaciones

#### 1. Documento en estado in_progress

```go
if doc.Status != "in_progress" {
    return ErrDocumentNotInProgress // 422: "document must be in in_progress status to publish"
}
```

#### 2. Secciones requeridas con contenido — BLOQUEANTE

Cada seccion con `required: true` en `config.coord_doc_sections` debe tener `value` no vacio (trim whitespace). Si faltan → 422 con lista.

#### 3. Sub-topics asignados a materia — WARNING CONFIRMABLE (P13)

Recordar: un documento de coordinacion esta relacionado a un **topic padre** y las materias a **topics hijos** (sub-topics). Cada sub-topic debe estar asignado a al menos 1 materia.

Si hay sub-topics sin asignar y `force == false` → retornar warning:

```json
{
  "status": "warning",
  "message": "Te quedaron sub-topics sin asignar a ninguna materia",
  "warnings": [
    {
      "type": "unassigned_subtopics",
      "details": [
        {"id": 5, "name": "Ecuaciones lineales"},
        {"id": 8, "name": "Funciones"}
      ]
    }
  ],
  "requires_confirmation": true
}
```

Si `force == true` → publicar a pesar de los warnings.

### Flujo completo

```go
func (uc *PublishDocumentUsecase) Execute(ctx context.Context, req PublishDocumentRequest) (*PublishResult, error) {
    doc, err := uc.docRepo.GetWithPreloads(ctx, req.OrgID, req.DocID)
    if err != nil {
        return nil, err
    }

    // Validacion 1: estado
    if doc.Status != "in_progress" {
        return nil, NewValidationError("document must be in in_progress status to publish")
    }

    // Validacion 2: secciones requeridas (BLOQUEANTE)
    emptySections, err := uc.validateRequiredSections(ctx, doc)
    if err != nil {
        return nil, err
    }
    if len(emptySections) > 0 {
        return nil, NewValidationError("empty required sections", "empty_required_sections", emptySections)
    }

    // Validacion 3: sub-topics asignados (WARNING confirmable)
    unassigned, err := uc.docRepo.GetUnassignedSubtopics(ctx, req.DocID)
    if err != nil {
        return nil, err
    }
    if len(unassigned) > 0 && !req.Force {
        return &PublishResult{
            Status:              "warning",
            RequiresConfirmation: true,
            Warnings: []Warning{
                {
                    Type:    "unassigned_subtopics",
                    Message: "Te quedaron sub-topics sin asignar a ninguna materia",
                    Details: unassigned,
                },
            },
        }, nil
    }

    // Publicar
    err = uc.docRepo.UpdateStatus(ctx, req.OrgID, req.DocID, "published")
    if err != nil {
        return nil, err
    }

    return &PublishResult{Status: "published"}, nil
}
```

### Endpoint

```
POST /api/v1/coordination-documents/:id/publish
{ "force": false }  // o true para confirmar a pesar de warnings
```

### Response de warning (requiere confirmacion)

```json
{
  "status": "warning",
  "message": "Te quedaron sub-topics sin asignar a ninguna materia",
  "warnings": [
    {
      "type": "unassigned_subtopics",
      "details": [
        {"id": 5, "name": "Ecuaciones lineales"}
      ]
    }
  ],
  "requires_confirmation": true
}
```

### Response de error bloqueante (422)

```json
{
  "error": "validation_failed",
  "message": "No se puede publicar el documento",
  "details": {
    "empty_required_sections": [
      "Eje problematico",
      "Estrategia metodologica"
    ]
  }
}
```

## Notas

- Las validaciones de secciones se ejecutan primero (bloqueantes) antes de las de sub-topics (warnings)
- El warning de sub-topics no bloquea: el coordinador puede publicar igual con `force: true`
- La query de sub-topics no asignados cruza `coord_doc_topics` vs `coord_doc_subject_topics`
- Una vez publicado, las secciones siguen editables pero las clases son inmutables (P12)
