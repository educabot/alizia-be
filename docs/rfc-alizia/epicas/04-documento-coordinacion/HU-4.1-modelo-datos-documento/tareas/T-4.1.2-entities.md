# T-4.1.2: Entities Go del documento de coordinación

**Historia:** HU-4.1 — Modelo de datos del documento
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Crear las entities Go con tags GORM para todas las tablas del documento de coordinación, incluyendo las nuevas tablas de chat messages (P15) y snapshots (P20). Cada entity debe tener las relaciones necesarias para que GORM pueda hacer preloads anidados del arbol completo.

## Implementación

Archivo: `src/core/entities/coordination_document.go`

```go
package entities

import (
    "time"

    "gorm.io/datatypes"
)

type CoordinationDocument struct {
    ID             uint              `json:"id" gorm:"primaryKey"`
    OrganizationID uint              `json:"organization_id"`
    Name           string            `json:"name"`
    AreaID         uint              `json:"area_id"`
    StartDate      time.Time         `json:"start_date" gorm:"type:date"`
    EndDate        time.Time         `json:"end_date" gorm:"type:date"`
    Status         string            `json:"status" gorm:"type:coord_doc_status;default:'pending'"`
    Sections       datatypes.JSON    `json:"sections" gorm:"type:jsonb;default:'{}'"`
    CreatedAt      time.Time         `json:"created_at"`
    UpdatedAt      time.Time         `json:"updated_at"`

    // Relations
    Area      *Area              `json:"area,omitempty" gorm:"foreignKey:AreaID"`
    DocTopics []CoordDocTopic    `json:"doc_topics,omitempty" gorm:"foreignKey:CoordinationDocumentID"`
    Subjects  []CoordinationDocumentSubject `json:"subjects,omitempty" gorm:"foreignKey:CoordinationDocumentID"`
}

type CoordDocTopic struct {
    ID                      uint `json:"id" gorm:"primaryKey"`
    CoordinationDocumentID  uint `json:"coordination_document_id"`
    TopicID                 uint `json:"topic_id"`

    // Relations
    Topic *Topic `json:"topic,omitempty" gorm:"foreignKey:TopicID"`
}

type CoordinationDocumentSubject struct {
    ID                      uint   `json:"id" gorm:"primaryKey"`
    CoordinationDocumentID  uint   `json:"coordination_document_id"`
    SubjectID               uint   `json:"subject_id"`
    ClassCount              int    `json:"class_count"`
    Observations            string `json:"observations"`

    // Relations
    Subject       *Subject                `json:"subject,omitempty" gorm:"foreignKey:SubjectID"`
    SubjectTopics []CoordDocSubjectTopic  `json:"subject_topics,omitempty" gorm:"foreignKey:CoordDocSubjectID"`
    Classes       []CoordDocClass         `json:"classes,omitempty" gorm:"foreignKey:CoordDocSubjectID"`
}

type CoordDocSubjectTopic struct {
    ID               uint `json:"id" gorm:"primaryKey"`
    CoordDocSubjectID uint `json:"coord_doc_subject_id"`
    TopicID          uint `json:"topic_id"`

    // Relations
    Topic *Topic `json:"topic,omitempty" gorm:"foreignKey:TopicID"`
}

type CoordDocClass struct {
    ID               uint   `json:"id" gorm:"primaryKey"`
    CoordDocSubjectID uint  `json:"coord_doc_subject_id"`
    ClassNumber      int    `json:"class_number"`
    Title            string `json:"title"`
    Objective        string `json:"objective"`

    // Relations
    ClassTopics []CoordDocClassTopic `json:"class_topics,omitempty" gorm:"foreignKey:CoordDocClassID"`
}

type CoordDocClassTopic struct {
    ID             uint `json:"id" gorm:"primaryKey"`
    CoordDocClassID uint `json:"coord_doc_class_id"`
    TopicID        uint `json:"topic_id"`

    // Relations
    Topic *Topic `json:"topic,omitempty" gorm:"foreignKey:TopicID"`
}

// P15: Chat messages persistidos en backend
type CoordDocChatMessage struct {
    ID                      uint           `json:"id" gorm:"primaryKey"`
    CoordinationDocumentID  uint           `json:"coordination_document_id"`
    Role                    string         `json:"role"`        // "user", "assistant", "system"
    Content                 string         `json:"content"`
    ToolCalls               datatypes.JSON `json:"tool_calls" gorm:"type:jsonb"`
    CreatedAt               time.Time      `json:"created_at"`
}

// P20: Snapshots para undo (hasta 3 por documento)
type CoordDocSnapshot struct {
    ID                      uint           `json:"id" gorm:"primaryKey"`
    CoordinationDocumentID  uint           `json:"coordination_document_id"`
    Sections                datatypes.JSON `json:"sections" gorm:"type:jsonb"`
    Classes                 datatypes.JSON `json:"classes" gorm:"type:jsonb"`
    CreatedAt               time.Time      `json:"created_at"`
}
```

## Notas

- `Sections` usa `datatypes.JSON` de GORM — requiere import de `gorm.io/datatypes`
- `Status` se mapea al enum de Postgres con `gorm:"type:coord_doc_status"` — ahora con 4 valores: pending, in_progress, published, archived (P1/P2)
- Los campos de relaciones usan puntero (`*Area`, `*Topic`) para que sean nullable cuando no se hace preload
- Los slices de relaciones permiten preloads anidados sin puntero
- `CoordDocChatMessage`: `ToolCalls` es JSONB nullable — solo se populan cuando el LLM ejecuta tools
- `CoordDocSnapshot`: `Classes` almacena un JSON serializado de todas las clases y sus topics, para poder restaurar sin reconstituir relaciones individuales
- Las FKs de GORM (`foreignKey:CoordDocSubjectID`) deben coincidir exactamente con el campo del struct
