# T-4.1.1: Migración: tablas del documento de coordinación

**Historia:** HU-4.1 — Modelo de datos del documento
**Tipo:** DB
**Estado:** ⬜ Pendiente

---

## Descripción

Crear el enum `coord_doc_status` y todas las tablas del documento de coordinación: la tabla principal, las junction tables para topics a nivel documento, disciplinas, topics por disciplina, clases y topics por clase. Incluye también las tablas de chat messages (P15) y snapshots (P20). Este modelo normalizado reemplaza el JSONB monolítico de la POC.

## SQL (up)

```sql
-- Enum para estado del documento (P1/P2: incluye archived)
CREATE TYPE coord_doc_status AS ENUM ('pending', 'in_progress', 'published', 'archived');

-- Tabla principal del documento de coordinación
CREATE TABLE coordination_documents (
    id SERIAL PRIMARY KEY,
    organization_id INTEGER NOT NULL REFERENCES organizations(id),
    name VARCHAR(255) NOT NULL,
    area_id INTEGER NOT NULL REFERENCES areas(id),
    start_date DATE NOT NULL,
    end_date DATE NOT NULL,
    status coord_doc_status NOT NULL DEFAULT 'pending',
    sections JSONB NOT NULL DEFAULT '{}',
    created_at TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMP NOT NULL DEFAULT NOW()
);

-- Junction: documento ↔ topic (topics seleccionados a nivel documento)
CREATE TABLE coord_doc_topics (
    id SERIAL PRIMARY KEY,
    coordination_document_id INTEGER NOT NULL REFERENCES coordination_documents(id) ON DELETE CASCADE,
    topic_id INTEGER NOT NULL REFERENCES topics(id) ON DELETE CASCADE,
    UNIQUE(coordination_document_id, topic_id)
);

-- Disciplinas dentro del documento (doc ↔ subject + class_count)
CREATE TABLE coordination_document_subjects (
    id SERIAL PRIMARY KEY,
    coordination_document_id INTEGER NOT NULL REFERENCES coordination_documents(id) ON DELETE CASCADE,
    subject_id INTEGER NOT NULL REFERENCES subjects(id),
    class_count INTEGER NOT NULL CHECK (class_count >= 1),  -- P5: mínimo 1
    observations TEXT
);

-- Junction: disciplina en doc ↔ topic (topics asignados a una disciplina)
CREATE TABLE coord_doc_subject_topics (
    id SERIAL PRIMARY KEY,
    coord_doc_subject_id INTEGER NOT NULL REFERENCES coordination_document_subjects(id) ON DELETE CASCADE,
    topic_id INTEGER NOT NULL REFERENCES topics(id) ON DELETE CASCADE,
    UNIQUE(coord_doc_subject_id, topic_id)
);

-- Clases individuales por disciplina en el documento
CREATE TABLE coord_doc_classes (
    id SERIAL PRIMARY KEY,
    coord_doc_subject_id INTEGER NOT NULL REFERENCES coordination_document_subjects(id) ON DELETE CASCADE,
    class_number INTEGER NOT NULL,
    title VARCHAR(255),
    objective TEXT
);

-- Junction: clase ↔ topic (topics cubiertos en una clase)
CREATE TABLE coord_doc_class_topics (
    id SERIAL PRIMARY KEY,
    coord_doc_class_id INTEGER NOT NULL REFERENCES coord_doc_classes(id) ON DELETE CASCADE,
    topic_id INTEGER NOT NULL REFERENCES topics(id) ON DELETE CASCADE,
    UNIQUE(coord_doc_class_id, topic_id)
);

-- Historial de chat con Alizia (P15: persistir en backend)
CREATE TABLE coord_doc_chat_messages (
    id SERIAL PRIMARY KEY,
    coordination_document_id INTEGER NOT NULL REFERENCES coordination_documents(id) ON DELETE CASCADE,
    role VARCHAR(20) NOT NULL,        -- 'user', 'assistant', 'system'
    content TEXT NOT NULL,
    tool_calls JSONB,                 -- tool calls del assistant si aplica
    created_at TIMESTAMP NOT NULL DEFAULT NOW()
);

-- Snapshots para undo (P20: hasta 3 versiones)
CREATE TABLE coord_doc_snapshots (
    id SERIAL PRIMARY KEY,
    coordination_document_id INTEGER NOT NULL REFERENCES coordination_documents(id) ON DELETE CASCADE,
    sections JSONB NOT NULL,
    classes JSONB NOT NULL,           -- serialized array of all classes + topics
    created_at TIMESTAMP NOT NULL DEFAULT NOW()
);

-- Indexes
CREATE INDEX idx_coordination_documents_organization_id ON coordination_documents(organization_id);
CREATE INDEX idx_coordination_documents_area_id ON coordination_documents(area_id);
CREATE INDEX idx_coordination_documents_status ON coordination_documents(status);
CREATE INDEX idx_coord_doc_topics_document_id ON coord_doc_topics(coordination_document_id);
CREATE INDEX idx_coord_doc_topics_topic_id ON coord_doc_topics(topic_id);
CREATE INDEX idx_coord_doc_subjects_document_id ON coordination_document_subjects(coordination_document_id);
CREATE INDEX idx_coord_doc_subjects_subject_id ON coordination_document_subjects(subject_id);
CREATE INDEX idx_coord_doc_subject_topics_subject_id ON coord_doc_subject_topics(coord_doc_subject_id);
CREATE INDEX idx_coord_doc_classes_subject_id ON coord_doc_classes(coord_doc_subject_id);
CREATE INDEX idx_coord_doc_class_topics_class_id ON coord_doc_class_topics(coord_doc_class_id);
CREATE INDEX idx_coord_doc_chat_messages_document_id ON coord_doc_chat_messages(coordination_document_id);
CREATE INDEX idx_coord_doc_snapshots_document_id ON coord_doc_snapshots(coordination_document_id);
```

## SQL (down)

```sql
DROP TABLE IF EXISTS coord_doc_snapshots;
DROP TABLE IF EXISTS coord_doc_chat_messages;
DROP TABLE IF EXISTS coord_doc_class_topics;
DROP TABLE IF EXISTS coord_doc_classes;
DROP TABLE IF EXISTS coord_doc_subject_topics;
DROP TABLE IF EXISTS coordination_document_subjects;
DROP TABLE IF EXISTS coord_doc_topics;
DROP TABLE IF EXISTS coordination_documents;
DROP TYPE IF EXISTS coord_doc_status;
```

## Notas

- El orden de DROP es inverso al de CREATE para respetar FKs
- `ON DELETE CASCADE` en todas las junction tables y tablas dependientes: borrar un documento limpia todo su arbol, incluyendo chat y snapshots
- `sections` es JSONB con default `{}` — contenido dinámico definido por config de la org (ver HU-4.3)
- El enum `coord_doc_status` tiene **4 valores** (P1/P2): pending, in_progress, published, archived
- `class_count` tiene CHECK constraint `>= 1` (P5)
- `coord_doc_snapshots` no tiene UNIQUE constraint — se permiten hasta 3 por documento (la lógica de limpieza va en el backend)
- `coord_doc_chat_messages.tool_calls` almacena el JSON de tool calls del LLM para auditoría y replay
- `updated_at` necesitará un trigger o actualización manual desde el backend
