# T-4.6.1: Definicion de tools para function calling

**Historia:** HU-4.6 — Chat con Alizia
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripcion

Definir las herramientas (tools) de function calling que se envian a Azure OpenAI para que el chat de Alizia pueda modificar el documento de coordinacion mediante lenguaje natural. Set minimo de 3 tools (P16). Los tools disponibles se filtran segun el estado del documento (P12): en `published`, los tools de clases no estan disponibles.

## Implementacion

### Ubicacion

`src/core/usecases/coordination/chat_tools.go`

### Tools — Set minimo (P16 — Decision)

#### 1. update_section

Reemplaza el contenido completo de una seccion del documento.

```go
var UpdateSectionTool = openai.Tool{
    Type: "function",
    Function: &openai.FunctionDefinition{
        Name:        "update_section",
        Description: "Reemplaza el contenido completo de una seccion del documento de coordinacion. Usar cuando el usuario pide cambiar o reescribir una seccion entera.",
        Parameters: jsonschema.Definition{
            Type: "object",
            Properties: map[string]jsonschema.Definition{
                "section_key": {
                    Type:        "string",
                    Description: "Clave de la seccion a actualizar (ej: 'problem_edge', 'methodological_strategy')",
                },
                "content": {
                    Type:        "string",
                    Description: "Nuevo contenido completo de la seccion",
                },
            },
            Required: []string{"section_key", "content"},
        },
    },
}
```

#### 2. append_to_section

Agrega texto al final de una seccion existente, preservando el contenido actual.

```go
var AppendToSectionTool = openai.Tool{
    Type: "function",
    Function: &openai.FunctionDefinition{
        Name:        "append_to_section",
        Description: "Agrega texto al final de una seccion existente sin eliminar el contenido actual. Usar cuando el usuario pide agregar o complementar informacion a una seccion.",
        Parameters: jsonschema.Definition{
            Type: "object",
            Properties: map[string]jsonschema.Definition{
                "section_key": {
                    Type:        "string",
                    Description: "Clave de la seccion donde agregar contenido",
                },
                "content": {
                    Type:        "string",
                    Description: "Texto a agregar al final de la seccion",
                },
            },
            Required: []string{"section_key", "content"},
        },
    },
}
```

#### 3. update_class (unificado)

Edita titulo, objetivo y/o topics de una clase especifica. Unifica `update_class_title` y `update_class_topics` del RFC original en un solo tool.

```go
var UpdateClassTool = openai.Tool{
    Type: "function",
    Function: &openai.FunctionDefinition{
        Name:        "update_class",
        Description: "Edita una clase especifica del plan de clases. Puede cambiar titulo, objetivo y/o topics asignados. Usar cuando el usuario pide modificar una clase. NO disponible cuando el documento esta publicado.",
        Parameters: jsonschema.Definition{
            Type: "object",
            Properties: map[string]jsonschema.Definition{
                "class_id": {
                    Type:        "integer",
                    Description: "ID de la clase a actualizar",
                },
                "title": {
                    Type:        "string",
                    Description: "Nuevo titulo de la clase (opcional, omitir para no cambiar)",
                },
                "objective": {
                    Type:        "string",
                    Description: "Nuevo objetivo de la clase (opcional, omitir para no cambiar)",
                },
                "topic_ids": {
                    Type:        "array",
                    Items:       &jsonschema.Definition{Type: "integer"},
                    Description: "Lista de IDs de topics a asignar a la clase (opcional, omitir para no cambiar)",
                },
            },
            Required: []string{"class_id"},
        },
    },
}
```

### Listas de tools por estado (P12 — Decision)

```go
// Tools disponibles en in_progress (todos)
var InProgressChatTools = []openai.Tool{
    UpdateSectionTool,
    AppendToSectionTool,
    UpdateClassTool,
}

// Tools disponibles en published (solo secciones)
var PublishedChatTools = []openai.Tool{
    UpdateSectionTool,
    AppendToSectionTool,
}
```

### Seleccion de tools en el usecase

```go
func getToolsForStatus(status string) []openai.Tool {
    if status == "published" {
        return PublishedChatTools
    }
    return InProgressChatTools
}
```

### System prompt segun estado

Cuando el documento esta publicado, el system prompt incluye:

```
"El documento esta publicado. Las clases son inmutables — no podes modificarlas.
Solo podes editar secciones narrativas (eje problematico, estrategia, criterios, etc.).
Si el coordinador te pide editar una clase, explicale que las clases no se pueden
modificar en un documento publicado y que el teacher puede editarlas en su planificacion."
```

## Notas

- 3 tools en vez de 4 del RFC original: `update_class_title` y `update_class_topics` se unificaron en `update_class` con campos opcionales
- Los `class_id` que el LLM usa deben corresponder a IDs reales de `coord_doc_classes`. El system prompt incluye los IDs disponibles
- No incluir tools de eliminacion o creacion de clases (P10)
- La re-generacion de secciones se hace via `update_section` con contenido nuevo generado por el propio LLM — no hay un tool separado de "regenerar" (P7)
- Best-effort: si un tool call falla, se reintenta 1 vez. Si sigue fallando, se reporta el error pero los demas tools se ejecutan (P18)
