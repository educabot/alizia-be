# T-4.6.3: Endpoints de chat con documento

**Historia:** HU-4.6 — Chat con Alizia
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripcion

Implementar los endpoints para el chat con Alizia: enviar mensajes y recuperar historial persistido (P15). El request se simplifica respecto al RFC original — ya no recibe history porque se carga del backend.

## Implementacion

### Endpoints

```
POST /api/v1/coordination-documents/:id/chat        — enviar mensaje
GET  /api/v1/coordination-documents/:id/chat/history — recuperar historial
```

### POST /chat — Request body

```json
{
  "message": "Cambia el titulo de la clase 3 de Fisica a 'Introduccion a la cinematica'"
}
```

| Campo | Tipo | Requerido | Descripcion |
|-------|------|-----------|-------------|
| `message` | string | Si | Mensaje actual del usuario |

Ya no se envía `history` — el backend lo carga de `coord_doc_chat_messages` automaticamente (P15).

### POST /chat — Response exitosa (200)

```json
{
  "message": "Listo! Cambie el titulo de la clase 3 de Fisica a 'Introduccion a la cinematica'.",
  "actions": [
    {
      "tool": "update_class",
      "result": "Clase 42 actualizada",
      "success": true
    }
  ]
}
```

### POST /chat — Response con warning (documento published)

Si el coordinador pide editar una clase en un documento publicado, Alizia responde en texto (no hay tool disponible):

```json
{
  "message": "No puedo modificar las clases porque el documento esta publicado. Las clases son inmutables una vez publicado. El teacher puede editarlas en su planificacion.",
  "actions": []
}
```

### GET /chat/history — Response

```json
{
  "messages": [
    {"id": 1, "role": "user", "content": "Hola Alizia", "created_at": "2026-04-23T10:00:00Z"},
    {"id": 2, "role": "assistant", "content": "Hola! En que puedo ayudarte?", "tool_calls": null, "created_at": "2026-04-23T10:00:05Z"}
  ],
  "total": 2
}
```

Query params opcionales: `limit` (default 50), `offset` (default 0).

### Responses de error

| Codigo | Situacion |
|--------|-----------|
| `400` | Body invalido o `message` vacio |
| `403` | Documento en estado `archived` o `pending` (sin ediciones previas), o usuario sin permisos |
| `404` | Documento no encontrado |
| `503` | Error de IA despues de reintento (P17) |

### Handler POST

```go
func (h *Handler) ChatWithDocument(c echo.Context) error {
    docID, err := strconv.Atoi(c.Param("id"))
    if err != nil {
        return echo.NewHTTPError(400, "invalid document id")
    }

    var req ChatRequest
    if err := c.Bind(&req); err != nil {
        return echo.NewHTTPError(400, "invalid request body")
    }

    if strings.TrimSpace(req.Message) == "" {
        return echo.NewHTTPError(400, "message is required")
    }

    orgID := getOrgIDFromContext(c)

    result, err := h.chatUsecase.Execute(
        c.Request().Context(),
        orgID,
        uint(docID),
        req.Message,
    )
    if err != nil {
        if errors.Is(err, ErrForbidden) {
            return echo.NewHTTPError(403, err.Error())
        }
        if errors.Is(err, ErrServiceUnavailable) {
            return echo.NewHTTPError(503, err.Error())
        }
        return echo.NewHTTPError(500, "error processing chat message")
    }

    return c.JSON(200, result)
}
```

### Handler GET history

```go
func (h *Handler) GetChatHistory(c echo.Context) error {
    docID, err := strconv.Atoi(c.Param("id"))
    if err != nil {
        return echo.NewHTTPError(400, "invalid document id")
    }

    limit, _ := strconv.Atoi(c.QueryParam("limit"))
    offset, _ := strconv.Atoi(c.QueryParam("offset"))
    if limit <= 0 { limit = 50 }

    orgID := getOrgIDFromContext(c)

    // Verificar acceso al documento
    _, err = h.docRepo.GetByID(c.Request().Context(), orgID, uint(docID))
    if err != nil {
        return echo.NewHTTPError(404, "document not found")
    }

    messages, err := h.chatRepo.GetHistory(c.Request().Context(), uint(docID), limit, offset)
    if err != nil {
        return echo.NewHTTPError(500, "error loading chat history")
    }

    total, _ := h.chatRepo.GetMessageCount(c.Request().Context(), uint(docID))

    return c.JSON(200, map[string]interface{}{
        "messages": messages,
        "total":    total,
    })
}
```

### Tipos del request

```go
type ChatRequest struct {
    Message string `json:"message" validate:"required"`
}
```

### Registro en router

En `mapping.go`:

```go
docs := v1.Group("/coordination-documents")
docs.POST("/:id/chat", handler.ChatWithDocument, RequireRole("coordinator"))
docs.GET("/:id/chat/history", handler.GetChatHistory, RequireRole("coordinator"))
```

### Validaciones

1. **Autenticacion**: usuario debe estar autenticado
2. **Rol**: solo coordinadores pueden usar el chat (`RequireRole("coordinator")`)
3. **Pertenencia**: el documento debe pertenecer a la organizacion del usuario
4. **Estado**: in_progress o published. Archived/pending reciben 403. Pending se auto-transiciona a in_progress (P1)

## Notas

- El request se simplificó: solo `message`, sin `history` (P15)
- El backend es stateful respecto al chat — persiste y carga el historial automaticamente
- El GET history es util para el frontend al abrir/recargar la vista del chat
- La auto-compactacion es transparente: el frontend no necesita saber si el historial fue compactado
- Loguear errores de Azure OpenAI internamente pero retornar 503 con mensaje generico
