# T-0.1.5: Dockerfile + docker-compose local

**Historia:** HU-0.1 — Setup del proyecto
**Tipo:** Infra
**Estado:** ⬜ Pendiente

---

## Descripción

Crear Dockerfile multi-stage para producción y docker-compose para desarrollo local.

## Dockerfile

```dockerfile
# Build stage
FROM golang:1.22-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 go build -o /alizia-api ./cmd

# Run stage
FROM alpine:3.19
RUN apk --no-cache add ca-certificates
WORKDIR /app
COPY --from=builder /alizia-api .
EXPOSE 8080
CMD ["./alizia-api"]
```

## docker-compose.yml

```yaml
services:
  db:
    image: postgres:16
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: alizia
    ports:
      - "5432:5432"
    volumes:
      - pgdata:/var/lib/postgresql/data

volumes:
  pgdata:
```

## Notas

- El servicio Go se corre local con `go run ./cmd` (no en Docker para hot-reload)
- docker-compose solo levanta PostgreSQL para desarrollo
- El Dockerfile se usa para Railway y para testing local del build
