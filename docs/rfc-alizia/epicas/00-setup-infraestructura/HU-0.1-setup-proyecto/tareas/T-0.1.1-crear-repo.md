# T-0.1.1: Crear repo con estructura de directorios

**Historia:** HU-0.1 — Setup del proyecto
**Tipo:** Infra
**Estado:** ⬜ Pendiente

## Descripción

Crear el repositorio `alizia-api` en GitHub (org educabot) con la estructura de directorios de Clean Architecture definida en el RFC.

## Estructura esperada

```
alizia-api/
├── cmd/
│   ├── main.go              # Entrypoint
│   ├── dependencies.go      # Wire manual (o split en repositories.go, usecases.go, handlers.go)
│   └── routes.go            # Registro de rutas
├── internal/
│   ├── admin/               # Módulo: orgs, areas, subjects, topics, courses
│   │   ├── entities/
│   │   ├── providers/
│   │   ├── usecases/
│   │   ├── handlers/
│   │   └── repositories/
│   ├── coordination/        # Módulo: documentos, wizard, secciones
│   ├── teaching/            # Módulo: lesson plans, momentos
│   ├── resources/           # Módulo: fonts, tipos, recursos
│   └── ai/                  # Módulo: Azure OpenAI, chat, function calling
├── db/
│   └── migrations/          # SQL migrations (.up.sql / .down.sql)
├── config/
│   └── config.go            # App config (embeds team-ai-toolkit/config.BaseConfig)
├── Dockerfile
├── docker-compose.yml
├── .github/
│   └── workflows/
├── .golangci.yml
├── go.mod
├── go.sum
└── README.md
```

## Notas

- Solo crear la estructura de carpetas, no implementar lógica
- `cmd/` split de DI: si el archivo crece, separar en `repositories.go`, `usecases.go`, `handlers.go`
- Módulos internos se crean vacíos, se llenan en las fases correspondientes
