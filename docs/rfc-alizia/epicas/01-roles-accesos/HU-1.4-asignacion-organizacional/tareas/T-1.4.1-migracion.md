# T-1.4.1: Migración: area_coordinators + course_subjects

**Historia:** HU-1.4 — Asignación organizacional
**Tipo:** Backend / DB
**Estado:** ⬜ Pendiente

---

## Descripción

Crear la migración con las tablas de asignación organizacional: qué coordinadores manejan qué áreas y qué docentes dictan qué materia en qué curso.

## SQL (up)

```sql
-- Area coordinators (M2M: users ↔ areas)
CREATE TABLE area_coordinators (
    id SERIAL PRIMARY KEY,
    area_id INTEGER NOT NULL REFERENCES areas(id) ON DELETE CASCADE,
    user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    created_at TIMESTAMP NOT NULL DEFAULT NOW(),
    UNIQUE(area_id, user_id)
);

-- Course subjects (curso + materia + docente asignado)
CREATE TABLE course_subjects (
    id SERIAL PRIMARY KEY,
    course_id INTEGER NOT NULL REFERENCES courses(id) ON DELETE CASCADE,
    subject_id INTEGER NOT NULL REFERENCES subjects(id) ON DELETE CASCADE,
    teacher_id INTEGER NOT NULL REFERENCES users(id),
    organization_id INTEGER NOT NULL REFERENCES organizations(id),
    school_year INTEGER NOT NULL DEFAULT EXTRACT(YEAR FROM NOW()),
    created_at TIMESTAMP NOT NULL DEFAULT NOW(),
    UNIQUE(course_id, subject_id, school_year)
);

-- Indexes
CREATE INDEX idx_area_coordinators_area_id ON area_coordinators(area_id);
CREATE INDEX idx_area_coordinators_user_id ON area_coordinators(user_id);
CREATE INDEX idx_course_subjects_course_id ON course_subjects(course_id);
CREATE INDEX idx_course_subjects_teacher_id ON course_subjects(teacher_id);
CREATE INDEX idx_course_subjects_org_id ON course_subjects(organization_id);
```

## SQL (down)

```sql
DROP TABLE IF EXISTS course_subjects;
DROP TABLE IF EXISTS area_coordinators;
```

## Notas

- `area_coordinators` es una tabla join pura (M2M) — un coordinador puede manejar varias áreas y un área puede tener varios coordinadores
- `course_subjects` incluye `organization_id` para facilitar queries multi-tenant sin joins adicionales
- `school_year` permite reutilizar la misma estructura año a año
- `UNIQUE(course_id, subject_id, school_year)` impide asignar la misma materia dos veces al mismo curso en el mismo año
- Esta migración depende de que `areas`, `subjects` y `courses` ya existan (Épica 3)
