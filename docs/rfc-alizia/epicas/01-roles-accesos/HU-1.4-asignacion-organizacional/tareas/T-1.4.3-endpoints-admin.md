# T-1.4.3: Endpoints admin de asignación

**Historia:** HU-1.4 — Asignación organizacional
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Implementar los endpoints admin para gestionar asignaciones de coordinadores a áreas y docentes a curso-materia.

## Endpoints

### Asignar coordinador a área

```
POST /api/v1/areas/:id/coordinators
Roles: admin
```

**Request:**
```json
{
    "user_id": 3
}
```

**Response 201:**
```json
{
    "id": 1,
    "area_id": 2,
    "user_id": 3,
    "created_at": "2026-01-15T10:00:00Z"
}
```

**Errores:**
- 404 — Área o usuario no existe
- 409 — Ya está asignado (`already_assigned`)
- 403 — No es admin

### Remover coordinador de área

```
DELETE /api/v1/areas/:id/coordinators/:user_id
Roles: admin
```

**Response:** 204 No Content

### Asignar docente a curso-materia

```
POST /api/v1/course-subjects
Roles: admin
```

**Request:**
```json
{
    "course_id": 1,
    "subject_id": 5,
    "teacher_id": 7,
    "school_year": 2026
}
```

**Response 201:**
```json
{
    "id": 1,
    "course_id": 1,
    "subject_id": 5,
    "teacher_id": 7,
    "organization_id": 1,
    "school_year": 2026,
    "created_at": "2026-01-15T10:00:00Z"
}
```

**Errores:**
- 404 — Curso, materia o docente no existe
- 409 — Materia ya asignada en ese curso/año (`already_assigned`)
- 403 — No es admin

### Listar asignaciones de un curso

```
GET /api/v1/courses/:id/subjects
Roles: todos (autenticados)
```

**Response 200:**
```json
[
    {
        "id": 1,
        "subject": { "id": 5, "name": "Matemáticas" },
        "teacher": { "id": 7, "name": "Prof. García" },
        "school_year": 2026
    }
]
```

## Implementación

```go
// Registrar rutas en cmd/routes.go
adminOnly := middleware.RequireRole("admin")

api.POST("/areas/:id/coordinators", webgin.Adapt(h.Admin.AssignCoordinator.Handle(), adminOnly))
api.DELETE("/areas/:id/coordinators/:user_id", webgin.Adapt(h.Admin.RemoveCoordinator.Handle(), adminOnly))
api.POST("/course-subjects", webgin.Adapt(h.Admin.CreateCourseSubject.Handle(), adminOnly))
api.GET("/courses/:id/subjects", webgin.Adapt(h.Admin.ListCourseSubjects.Handle()))
```

## Notas

- `organization_id` en `course_subjects` se toma del context (JWT claim), no del request body
- Validar que el `user_id` a asignar pertenezca a la misma organización
- Validar que el usuario tenga el rol apropiado (coordinador para áreas, teacher para course-subjects)
