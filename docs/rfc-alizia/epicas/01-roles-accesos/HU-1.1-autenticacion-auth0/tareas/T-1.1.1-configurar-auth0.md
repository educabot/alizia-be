# T-1.1.1: Configurar Auth0 tenant

**Historia:** HU-1.1 — Autenticación con Auth0
**Tipo:** Infra
**Estado:** ⬜ Pendiente

---

## Descripción

Configurar el tenant de Auth0 para Alizia v2. Usar el mismo sistema que tich-cronos.

## Pasos

1. Crear API en Auth0 dashboard:
   - Name: `alizia-api`
   - Identifier (audience): `https://api.alizia.educabot.com` (o el que defina el equipo)
   - Signing Algorithm: RS256
2. Configurar application (SPA) para el frontend React
3. Crear connection database (email + password)
4. Configurar custom claims en Auth0 Rules/Actions para incluir en el JWT:
   - `org_id` — organization ID del usuario
   - `roles` — array de roles del usuario (teacher, coordinator, admin)
5. Obtener domain y audience para staging

## Variables de entorno resultantes

```
AUTH0_DOMAIN=educabot-staging.auth0.com
AUTH0_AUDIENCE=https://api.alizia.educabot.com
```

## Notas

- El custom claim de `org_id` es CRÍTICO para multi-tenancy — viene del JWT, no del request
- Los roles también vienen del JWT como claim custom
- El frontend obtiene el token via Auth0 SPA SDK y lo envía como Bearer token
